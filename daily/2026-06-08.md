## 笔记与思考

### 关键概念

#### 1. 工厂模式是 Hackathon 评审的"安全开关"
`caw_factory.py` 的核心价值不是技术炫技，而是解决了一个真实的评审场景问题：

```
评委环境 → 无 Cobo SDK / 无 API Key / 无真实钱包
项目演示 → 必须能跑起来
```

工厂模式让 `CAW_MODE=mock` 时零依赖启动（评委 3 秒看到效果），`CAW_MODE=real` 时一行切换对接真实链上资金。这是"可评审性"的关键设计。

#### 2. 国际化不是翻译，是架构决策
今天将 222 条硬编码英文文本全部抽离到 `locales/en.json` 和 `locales/zh.json`，这个决策的深层影响：
- **评审体验**：中文评委能用自己的语言理解产品（降低认知负荷）
- **代码可维护性**：文本集中管理，改文案不动组件
- **扩展性**：新增语言只需加 JSON 文件，零代码改动
- **关键细节**：`i18n.on('languageChanged')` 同步 `<html lang>`，让 CSS `:lang(zh)` 排版规则自动生效——这是视觉层和语言层的联动设计

#### 3. 中文排版是"隐形竞争力"
大多数 Web3 项目直接套用英文排版规则（line-height 1.5、font-weight 700、letter-spacing normal），导致中文显示拥挤、粗体过黑、长词溢出。

今天的排版优化覆盖了三个维度：
- **字体栈**：`Noto Sans SC → PingFang SC → Hiragino Sans GB → Microsoft YaHei → Source Han Sans SC`
- **度量参数**：行高 1.75（vs 英文 1.5）、标题字重 500-600（vs 英文 700）、字距 0.01-0.02em
- **暗色补偿**：`html.dark[lang="zh"]` 提升字重和行高，补偿光晕效应

评委不会主动说"这个中文排版很好"，但会潜意识觉得"这个项目很专业"。

#### 4. Cooldown 是时间风控的两种表达
Pact 的时间窗口有两种互斥模式：
- `time_window`：固定起止日期（如 2026-06-01 → 2026-06-30）
- `cooldown_hours`：每次支付后强制冷却 N 小时（如 24h）

之前 UI 只支持第一种，今天补齐了第二种的显示逻辑。这是 Threat Simulator 中"时间维度攻击"（rapid-fire 高频攻击）的防御基础。

### 今日收获

- **工厂模式 = 可评审性**：Hackathon 评审不会配置你的环境。Mock/Real 工厂让项目在任何环境下 3 秒启动，这是"可被评分"的前提。
- **国际化是评审视角的扩展**：中英文切换不是炫技，而是让不同背景的评委都能用自己的语言理解产品。222 条翻译看似繁重，但完成后代码的文本可维护性大幅提升。

- **html lang 同步是 i18n 的隐藏关卡**：只做翻译不做排版联动，等于只做了半套国际化。`languageChanged` 事件同步 `<html lang>` 让 CSS 选择器自动生效，是语言层和视觉层的正确连接方式。
