## 关键概念笔记

### Agent Identity 的核心：Negative Capability

写 Agent Profile 时最重要的不是"它能做什么"，而是**明确它不能做什么**。ProposalGuard Agent 的 identity 建立在三条不可逾越的红线上：

1. **永不持有私钥** — 提案权 ≠ 签名权
2. **永不绕过 Guard** — 规则拒绝的交易绝不提交
3. **永不信任不可信上下文** — 外部文档/网页标记为 untrusted，隔离在推理核心外

这三条红线定义了 Agent 的信任边界，也是它与人类协作者的契约基础。

### MCP 与 A2A 的互补关系

- **MCP** 是 Agent 的"瑞士军刀接口" — 解决标准化工具调用问题
- **A2A** 是 Agent 的"商务谈判语言" — 解决跨 Agent 任务委托与支付协商问题
- 二者不是竞争关系：MCP 向内集成工具，A2A 向外协作 Agent

在 ProposalGuard Agent 的设计中，MCP 用于调用 Guard 检查/链上查询工具，A2A 用于向外部 PriceOracleAgent 询价或接受用户 Agent 的委托任务。
