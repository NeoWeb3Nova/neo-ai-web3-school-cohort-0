## 关键发现与決策

### 发现

1. **v1 和 v2 的本质差别在于"状态化"——v1 是无状态的单次检查，v2 是有状态的会话管理**
   - v1 对 A5 Budget Exhaustion 无能为力，因为每次检查独立，不记忆历史
   - v2 通过 `session_spent` + `hourly_tx_count` + `consecutive_failures` 维护会话状态，才能拦截累计攻击

2. **A8 社会工程是所有技术防护的盲点**
   - 无论 v1 还是 v2，都无法防止 "请求用户手动升级 Pact 预算"
   - 唯一防护在治理层：24h timelock + multi-sig

3. **Mock 模式是 Demo 开发的加速器**
   - 无需等待网络、无需配置 API Key、无需跳转测试网
   - 10 秒内即可验证一次代码变更，CI 级别反馈速度

### 决策

- **保持 v2 为默认版本**：所有演示和 Demo 视频都展示 v2 的拦截能力
- **优先证明"代码可运行"而非"代码完美"**：在 6/12 提交截止前，可运行的模拟 Demo 比不可运行的"完美"更有价值
