## 今日发现：代码资产的关键不一致

### 1. Pact Config 字段名不一致

`pact-config.json` 使用 camelCase：
- `allowedContracts`, `allowedNetworks`, `allowedFunctions`
- `denyContracts`, `denyFunctions`
- `createdAt`, `expiresAt`

而 Python 代码使用 snake_case：
- `allowed_contracts`, `allowed_networks`
- `createdAt`, `expiresAt` (混合)

**影响**：运行时会 KeyError，必须统一。

### 2. Pact 时间窗口已过期

`pact-config.json` 中 `expiresAt: "2026-05-31T00:00:00Z"` — 今天 6/1 已经过期，任何测试都会 `time_expired`。

### 3. v2 安全逻辑分散在两个文件

- `threat-model-simulator.py` 有完整的 v2 `check_pact()`（perTxMax + dailyAccumulated + hourlyFrequency + reputation + consecutiveFailures + autoRevoke）
- `pseudo-agent-client.py` 只有 v1 简化版（budget + scope + network + time）

**决策**：将 v2 逻辑合并回 `pseudo-agent-client.py`，`threat-model-simulator.py` 专注于攻击场景编排和对比输出。

### 4. Server 端依赖 `x402` Python SDK 不存在

`pseudo-server.py` 导入了 `from x402 import ...`，但环境中未安装。需要：
- 方案 A：pip install x402（如果存在包）
- 方案 B：用 FastAPI 手写 mock x402 中间件，让 client 能端到端跑通
- 方案 C：将 server 端降级为纯模拟，重点放在 client + threat model
