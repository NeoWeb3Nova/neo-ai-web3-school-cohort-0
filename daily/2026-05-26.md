## 笔记与思考

### 关键概念

#### 1. 方向决策框架（帕累托聚焦）

如果今天只做一件事：**确定 Hackathon 方向**。

三选一的判断标准：

| 维度 | Agentic Commerce | AI Security | 模块 C 延伸（治理自动化） |
|------|-----------------|-------------|------------------------|
| **已有基础** | 模块 C 原型 + Escrow 状态机 | 概念卡片较多，代码原型少 | 模块 C 原型 + Governance AI 章节 |
| **Demo 效果** | 最直观（交易、支付、 escrow 可见） | 偏抽象（检测、审计、策略） | 中等（提案摘要、风险标记） |
| **4 周可行** | 中高（Escrow + Agent Profile + Payment） | 中（检测规则 + 报告生成 + 人工复核） | 高（基于模块 C 快速迭代） |
| **差异化** | 赛道热门，需突出 escrow/verification 创新 | 赛道较新，容易做出差异化 | 需求真实，但展示性弱于 Commerce |

**决策建议**：
- 若追求 **Demo 冲击力和评委理解度** → Agentic Commerce
- 若追求 **技术深度和差异化** → AI Security
- 若追求 **4 周内完成度和可控性** → 模块 C 延伸（治理自动化）

#### 2. 5W 技术拆解模板

方向确定后，把 5W 填进下表：

| 问题 | 人/实体 | 技术组件 | 链上/链下 | AI/人工 |
|------|---------|----------|-----------|---------|
| **谁发起任务** | | | | |
| **谁执行** | | | | |
| **谁付款** | | | | |
| **谁验证** | | | | |
| **谁承担风险** | | | | |

示例（Agentic Commerce 方向）：
- 谁发起任务：用户（链下 Intent）或 AI Agent（链下识别需求）
- 谁执行：Seller Agent 生成交付物 / Buyer Agent 验收
- 谁付款：Buyer 钱包 → Escrow 合约（链上锁定）
- 谁验证：Evaluator（链下 AI 或链上规则）或人工仲裁
- 谁承担风险：Buyer（资金锁定风险）、Seller（交付不通过风险）、平台（声誉风险）
