## 模块 F 核心产出

### 威胁模型关键发现

1. **配置≠实现**：v1 Pact 的 `perTransactionMax: $5` 在配置中存在，但 `check_pact()` 实现未检查该字段 — 攻击 #2（$50 超量定价）被 v1 无条件通过
2. **单笔安全≠累计安全**：10 × $4.99 可在数分钟内耗尽 $50 总预算，v1 无任何频率/累计拦截 — v2 引入 `maxDailyAccumulated` + `maxHourlyFrequency` 后第 3 笔起拦截
3. **Prompt Injection 可被硬性边界拦截**：即使 LLM 被注入诱导转账给攻击者，只要目标地址不在 `allowedContracts`，Pact 范围检查会在签名前拦截
4. **社会工程学是终极盲区**：诱导用户手动升级 Pact 预算的攻击无法被 Agent 内部策略拦截 — 需要 UI 冷静期 + 多签治理
5. **反脆弱设计**：`pauseOnConsecutiveFailures: 2` + `autoRevokeSessionOnTimeout: true` 让系统在受压时自动收紧权限

### 攻击拦截率对比

| Attack | v1 Result | v2 Result |
|--------|-----------|-----------|
| A1 Prompt Injection → Unauthorized Transfer | 🛡️ Blocked (scope) | 🛡️ Blocked + Alert |
| A2 Forged Return → Overpriced | ⚠️ **FULL BYPASS** | 🛡️ Blocked (perTxMax) |
| A5 Budget Exhaustion → 10×$4.99 | ⚠️ **FULL BYPASS** | 🛡️ Blocked (daily+freq) |
| **拦截率** | **62.5% (5/8)** | **87.5% (7/8)** |

### v2 安全增强 Pact 配置

基于模拟结果，升级了 Pact 配置：
- `maxDailyAccumulated`: $10（低于总预算 $50，防止单日耗尽）
- `maxHourlyFrequency`: 2（滑动窗口频率限制）
- `onlyPreviouslyInteractedContracts`: true（首次交互需人工确认）
- `reputationMinScore`: 4.5（服务方声誉门槛）
- `pauseOnConsecutiveFailures`: 2（连续失败自动暂停）
- `autoRevokeSessionOnTimeout`: true（超时自动吊销 Session Key）

### 三个方向的当前准备度评估

| 维度 | 方向 A: Agentic Commerce | 方向 B: AI Security | 方向 C: 治理自动化 |
|------|-------------------------|--------------------|--------------------|

| **可演示性** | ⭐⭐⭐ 高（Agent 自动购买内容并支付，有明确前后端） | ⭐⭐ 中（攻击模拟脚本可运行，但防御系统需额外实现） | ⭐⭐ 中（治理流程慢节奏，Agent 提效难即时量化） |

| **周末可推进度** | ⭐⭐⭐ 高（伪代码→可运行脚本，测试网实验） | ⭐⭐ 中（需实现 Guard/Policy 运行时） | ⭐ 低（需选定具体 DAO 协议并研究规则） |

### 方向锁定：**Agentic Commerce + Security Layer（方向 A 为主，方向 B 为安全增强）**

**决策理由：**

1. **代码资产最完整**：x402 + CAW 实验已有架构设计、伪代码、Pact 配置、威胁模拟器，周末两天可直接推进到可运行 Demo
2. **差异化可被 Security 放大**：纯支付 Agent 不够差异化，但「带多层安全策略的自主支付 Agent」是独特卖点 — Threat Model 和 Guard 层是今天刚完成的核心资产
3. **课程模块自然融合**：Module A（问题地图）→ Module B（支付流）→ Module D（钱包权限）→ Module F（威胁模型）全部串联在这个方向里
4. **反脆弱设计可落地**：v2 Pact 的 `pauseOnConsecutiveFailures` 和 `autoRevokeSessionOnTimeout` 可以直接集成到 Demo 中展示「攻击时自动收紧权限」
5. **演示效果好**：可以现场演示攻击 #5（预算耗尽攻击）被 v2 拦截的过程，有明确的 before/after 对比

### 风险评估

- 用户信任门槛：「AI 自动花我的钱」需要清晰的 Budget + Quote + Receipt 三层展示 — 已有 x402 设计
- 支付流程复杂：需要整合 x402 Facilitator + CAW + 测试网 — 但已有伪代码，周末可推进
- 演示依赖测试网可用性：Base Sepolia 偶有拥堵 — 准备本地模拟 fallback

### 备选方案

如果周末测试网实验遇到不可逾越的阻塞，fallback 为：**纯 AI Security 方向** — 以 `threat-model-simulator.py` 为核心，做一个「Agent 安全策略沙盒」，让用户可配置 Pact 规则并观察不同攻击下的拦截结果。
