## 笔记与思考

### 关键概念

1. **Machine Payment 核心原则（Week 2 交叉基础）**
   - 预算先于执行：没有预算边界，就没有安全自动支付
   - 报价必须可比较：Agent 要知道价格、币种、有效期、服务范围和退款条件
   - 收据必须可验证：付款后要能证明付给谁、为什么付、交付了什么
   - **对 Hackathon 的启发**：如果项目涉及 Agent 购买服务，必须显式设计 Budget + Quote + Receipt 三层，而不是让 Agent 直接调用 "transfer"

2. **AI Security 分层防护（交叉问题必备）**
   - 第一层：上下文隔离 — 把外部内容（网页、合约文档、API 返回）标记为 untrusted context，不能覆盖系统规则
   - 第二层：工具权限隔离 — 读工具和写工具分开，普通任务和高风险任务分开
   - 第三层：Guard 拦截 — 在交易真正发出前，用确定性规则检查目标地址、金额、方法、模拟结果
   - 第四层：审计日志 — 记录完整链路（模型看到什么 → 选择什么工具 → policy 是否通过 → 用户是否确认）
   - **关键认知**：不是让模型永不犯错，而是让模型犯错时无法直接造成不可接受损失

3. **Agent Wallet = 权限系统，不是签名按钮**
   - Agent 不直接控制主资产：模型生成交易草稿，但主私钥不进入 Agent 上下文
   - Session Key 是小钥匙，不是小号私钥：必须限制有效时间、调用方法、单笔金额、总额度、可操作资产类型
   - Policy 要变成系统能检查的规则：模糊的 "帮我低风险操作" 不适合直接变成链上权限
   - Guard 负责拒绝越界动作：Agent 生成候选动作，Guard 用确定性规则拦截

4. **Week 2 方向深化（基于 5W 框架）**
   - 无论选择 Agentic Commerce 还是 AI Security，都必须回答：谁发起任务 / 谁执行 / 谁付款 / 谁验证 / 谁承担风险
   - 如果方向是 "AI 生成提案 → 人工复核 → 链上执行"（模块 C 延伸），核心交叉点是：
     - Agent Wallet：给 Agent 一个「提案权」Session Key（只读/草稿），签名权保留在人类钱包
     - Machine Payment：如果 Agent 需要调用付费 API 获取数据，必须有 Budget + Quote 层
     - AI Security：Agent 读取的合约文档、网页内容必须经过 untrusted context 标注

**模块 C 预算层伪代码（基于 Machine Payment 概念）**
```python
class AgentBudgetPolicy:
    """Agent 支付权限的最小策略检查器"""
    def __init__(self, config: dict):
        self.daily_max_usdc = config.get("daily_max_usdc", 10.0)
        self.per_call_max_usdc = config.get("per_call_max_usdc", 1.0)
        self.allowed_tokens = config.get("allowed_tokens", ["USDC", "USDT"])
        self.spent_today = 0.0
        self.call_count_today = 0

    def check_quote(self, quote: dict) -> tuple[bool, str]:
        """检查报价是否合规"""
        amount = quote.get("amount", 0)
        token = quote.get("token", "")
        
        if token not in self.allowed_tokens:
            return False, f"Token {token} not in allowed list"
        if amount > self.per_call_max_usdc:
            return False, f"Amount {amount} exceeds per-call limit {self.per_call_max_usdc}"
        if self.spent_today + amount > self.daily_max_usdc:
            return False, f"Daily budget exhausted: {self.spent_today}/{self.daily_max_usdc}"
        
        return True, "Quote approved"

    def record_payment(self, quote: dict, receipt: dict):
        """记录支付，生成可验证收据"""
        self.spent_today += quote["amount"]
        self.call_count_today += 1
        log_entry = {
            "timestamp": receipt["timestamp"],
            "quote_id": quote["quote_id"],
            "amount": quote["amount"],
            "token": quote["token"],
            "recipient": quote["recipient"],
            "tx_hash": receipt.get("tx_hash"),
            "chain": receipt.get("chain")
        }
        # 写入审计日志 / 锚定链上
        return log_entry
```

对比维度（5W 框架）：
| 维度 | Agentic Commerce | AI Security |
|------|------------------|-------------|
| Who initiates | 用户 or Agent 发现服务需求 | 用户 or 系统自动监控 |
| Who executes | Agent 协商、报价、支付 | Agent 检测、告警、拦截 |
| Who pays | 用户钱包 / Agent Budget | 安全基础设施方（或用户付监控费） |
| Who verifies | 服务方收据 + 链上结算记录 | Guard 规则 + 审计日志 |
| Who bears risk | 用户（支付失败、服务欺诈） | 用户（资产损失）+ 项目方（声誉） |
