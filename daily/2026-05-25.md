## 笔记与思考

### 关键概念

#### 1. Agent Identity —— 从临时会话到经济参与者

Agent Identity 不是起个名字，而是回答五个问题：
- **是谁**：agentId + owner（EOA / Smart Account / 多签 / DAO）
- **能做什么**：Capability（绑定 schema、价格、限制、风险等级）
- **怎么调用**：Service Endpoint（HTTPS / A2A / MCP / Webhook）
- **怎么验证**：Registry（链上锚点 + 更新历史）
- **历史可信吗**：DID/VC + 声誉记录

**对 Hackathon 的启发**：
- 如果做 Agentic Commerce：Agent Profile 必须机器可读，包含 payment、terms、capabilities，否则其他 Agent 无法自动发现服务
- 如果做治理自动化：Agent 的 Capability 要明确区分「只读分析」「生成草稿」「自动执行」三个风险等级，不能混为一谈
- **关键底线**：高价值 Agent 的 owner 不能是单个热钱包；profile / endpoint / 收款地址更新必须留下可验证记录

#### 2. Settlement & Escrow —— 把转账变成完整交易流程

核心状态机（最小 6 状态）：
```
Created → Funded → Delivered → Accepted → Released
              ↓         ↓            ↓
           Refunded  Disputed    Refunded
```

关键设计原则：
- **资金状态要清楚**：pending / locked / released / refunded / disputed
- **交付证明要可保存**：文件 hash、API 日志、模型输出签名、链上 event、TEE attestation
- **争议流程要提前设计**：谁有权发起、证据格式、裁决权、是否可申诉
- **验收要拆成可检查条件**：字段完整、测试通过、hash 匹配、预算未超

**对 Hackathon 的启发**：
- Agent-to-Agent 支付不能是裸 transfer，必须有任务 ID + quote + escrow + receipt
- Receipt 是声誉系统的输入：按时付款、按时交付、退款率都从这里提取
- **AI evaluator 不要独断**：建议「AI 初审 + challenge window + 人工复核」组合

#### 3. Governance AI —— 提高信息质量，不替代政治判断

治理 AI 的底线原则：
- **来源可追溯**：每条摘要回到原始提案、论坛、会议或链上记录
- **立场要分开**：事实、推断、观点、建议不能混为一谈
- **人保留决策权**：AI 辅助信息整理，投票和授权由人或明确规则完成

结构化摘要模板（可直接复用）：
```
1. 提案目标和背景
2. 预算金额、付款地址和里程碑
3. 支持理由和反对理由
4. 未回答问题
5. 相关历史提案或链上交易
6. 每条关键结论附来源链接
7. 明确标注「AI 没有替你做投票建议」
```

**对 Hackathon 的启发**：
- 方向 C（治理自动化）的核心交付物可以是一个「治理提案摘要生成器」，严格遵循上述 7 步模板
- Source Traceability 是最低要求：如果输出无法给出来源，必须标注为「推断」或「不确定」
- Plurality 理念：不要只给主流意见，要展示少数派担忧、利益相关者差异、可协商空间

**最小 Escrow 状态机（方向 A：Agentic Commerce —— 草稿）**
```python
class AgentEscrow:
    """最小 Agent-to-Agent 托管状态机"""
    STATES = ["Created", "Funded", "Delivered", "Accepted", "Released", "Refunded", "Disputed"]
    
    def __init__(self, task_id, buyer, seller, amount, token, deadline_minutes):
        self.task_id = task_id
        self.buyer = buyer
        self.seller = seller
        self.amount = amount
        self.token = token
        self.deadline = now() + deadline_minutes * 60
        self.state = "Created"
        self.delivery_proof = None
        self.evaluator_result = None
    
    def fund(self, by):
        assert by == self.buyer
        assert self.state == "Created"
        self.state = "Funded"
    
    def deliver(self, by, proof: dict):
        assert by == self.seller
        assert self.state == "Funded"
        assert now() < self.deadline
        self.delivery_proof = proof
        self.state = "Delivered"
    
    def accept(self, by):
        assert by == self.buyer
        assert self.state == "Delivered"
        self.state = "Released"
        # 触发 token transfer to seller
    
    def refund(self, by):
        # 可由 buyer（超时/失败）或系统（evaluator 不通过）触发
        assert self.state in ["Funded", "Delivered"]
        if self.state == "Funded" and now() > self.deadline:
            self.state = "Refunded"
        # ... 其他条件
    
    def dispute(self, by, evidence):
        assert by in [self.buyer, self.seller]
        assert self.state in ["Delivered", "Funded"]
        self.state = "Disputed"
        # 进入仲裁流程
```
