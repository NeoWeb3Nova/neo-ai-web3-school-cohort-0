## 笔记与思考

### 关键概念

#### 1. Hackathon 提交 = 产品化思维的检验
之前四周是功能开发，今天是把四周工作转化为"评委能 5 分钟看懂的价值"。关键不是代码量，而是：
- 痛点是否让评委 30 秒内共鸣（午夜惊魂）
- 方案是否一句话说清楚（给 AI Agent 发企业卡）
- 证据是否真实（测试网交易 hash、SDK 版本号、截图）

#### 2. Demo 视频是最高杠杆率的投入
5 分钟视频 = 评委 80% 的评分依据。脚本已经写好了（`demo/video-script.md` 300 行分镜），只需要按脚本录制：
- 片头标识（3 秒）
- 痛点 — 午夜惊魂（30 秒）
- 方案 — 虚拟员工卡（60 秒）
- 演示 — 正常流 + 异常拦截（90 秒）
- 技术 — 架构 + CAW 真实交互（60 秒）
- 收尾 — 团队 + 链接（17 秒）

#### 3. 提交前安全检查清单
必须确认无敏感信息泄露：
- `.env` 已在 `.gitignore` 中
- 无私钥 / mnemonic / API key 硬编码
- `mock_caw_client.py` 中的测试数据是 mock，不是真实私钥
- `demo/CAW-REAL-TEST-RESULT.md` 中的真实地址是测试网，非主网

   - 解决：使用 OBS / Screen Studio 按 `demo/video-script.md` 录制，或先用 CLI 录屏 `asciinema` / `terminalizer`

   - 解决：运行 `python src/run_demo.py full` 截取关键步骤，或复用 `demo/CAW-REAL-TEST-RESULT.md` 中的测试网交易数据

3. **README 中 Demo Video 链接仍为 TBD**
   - 需要视频上传后替换为实际链接（YouTube / Bilibili / Loom）
