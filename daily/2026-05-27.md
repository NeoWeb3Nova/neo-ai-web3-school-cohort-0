## 笔记与思考

### 关键概念

#### 1. Co-learning 日提问清单（建议携带）

如果你昨天已完成方向决策，今天可以带着以下问题去 Co-learning：

| 问题类型 | 示例 |
|---------|------|
| **方向验证** | "这个方向在 4 周内是否过于庞大/过于简单？" |
| **技术选型** | "Escrow 用现有协议（如 Safe/ENS）还是自研合约更合适？" |
| **Demo 设计** | "评委最看重什么：代码完整性、商业模式、还是技术创新？" |
| **资源推荐** | "有没有推荐的测试网 Faucet、模板仓库或 SDK？" |
| **组队建议** | "这个方向适合单兵还是组队？如果需要组队，缺什么角色？" |

#### 2. 反馈迭代模板

Co-learning 后，用以下格式记录反馈并迭代：

```
- 新增/删除了哪些技术组件
- 调整了什么假设或边界

【下一步验证】
- 用什么方式验证这个迭代是否有效
```

#### 3. 3 步最小 Demo 场景设计

无论选哪个方向，今天可以设计一个「3 步内可演示」的最小场景：

- **Agentic Commerce**：
  1. Buyer 发 Intent（"我要买一份智能合约审计报告"）
  2. Seller Agent 接单并交付（生成报告摘要 + 锁定到 Escrow）
  3. Evaluator 验证通过 → 释放资金

- **AI Security**：
  1. 用户提交一笔待签名交易
  2. AI Agent 分析风险并标记（高风险/中风险/低风险）
  3. 高风险 → 强制人工复核；低风险 → 自动通过

- **治理自动化**：
  1. 治理提案自动生成（基于链上数据摘要）
  2. 人工确认提案内容
  3. 签名提交到治理合约
