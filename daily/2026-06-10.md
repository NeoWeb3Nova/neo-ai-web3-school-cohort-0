## 笔记与思考

### 关键概念

#### 1. x402 是 Agent 支付的 HTTP 中间件协议
今日新增 `service_registry.py` 207 行，实现 x402 provider 注册与 ERC-8004 元数据解析。x402 的核心思想是：Agent 在发起 HTTP 请求时，如果服务端返回 `402 Payment Required`，Agent 自动从已授权的卡片中完成支付，无需人工干预。ERC-8004 提供标准化的元数据格式（`name` / `version` / `supportedChains` / `paymentMethods`），让 Agent 知道该向哪个 provider 请求支付。

**实现要点**：
- `service_registry.py` 维护 provider 注册表，支持动态加载 x402 兼容服务
- 后端新增 `/marketplace/providers` 和 `/marketplace/quote` 端点，返回 ERC-8004 元数据
- 前端 AgentConsole 调用 `cawApi.submitPayment()` 时，自动携带 x402 协议头

#### 2. AgentConsole 全面接入 Live Cards = 状态管理复杂度陡增
AgentConsole 页面从静态 mock 重构为 API 驱动，核心难点：
- 需要同时管理 `selectedCard`（当前选中的卡片）、`paymentRequest`（表单状态）、`loadingStates`（每个操作的独立 loading）
- 卡片列表用 `useEffect` 加载，但支付提交后需要局部刷新而非全量重载
- 未知状态（`EXPIRED`、`REJECTED`、后端新增状态）必须降级渲染，不能白屏

**解法**：新增 `web/src/utils/cardStatus.tsx`，把状态映射到颜色、标签、可用性三元组，组件层只读不判断。

#### 3. FastAPI + 异步 CAW SDK 的事件循环陷阱（再次命中）
`Fix CAW cards read path event loop issue`：在 async endpoint 里直接调用 SDK 的 async client，如果 SDK 内部使用了不同的 `aiohttp.ClientSession`，会触发 `RuntimeError: Event loop is closed`。这是之前已知问题（见 06-08 daily），今天在后端 read path 再次命中。

**解法**：read path 全部切换为 sync REST GET（`requests.get`），write path 保持 async SDK 调用但做 session 隔离。后续应统一封装 sync wrapper。

#### 4. Dashboard 图表 sizing warnings 的 root cause
Recharts 在容器尺寸未确定时渲染会报 `Warning: The width(0) and height(0) of chart should be greater than 0`。这不是数据问题，是 React 渲染时序问题：父容器 `div` 还没完成 layout，子图表已经 mount。

**解法**：给图表容器加 `min-h-[200px]` 和 `w-full`，确保 Recharts mount 时容器有确定的尺寸。AuditReport 页面同样处理。

### 问题与卡点

1. **AgentConsole 重构后表单状态丢失**
   - 原因：从 mock 数据切到 API 后，`selectedCard` 初始值为 null，表单在卡片加载前尝试渲染导致 crash
   - 解决：增加 `!cards.length` 时的 loading 骨架屏，表单在 `selectedCard` 确定后才渲染

2. **x402 供应商选择后保存丢失**
   - 原因：创建卡片时前端传了 `selectedVendors`，但后端 `CardResponse` 没有 `x402_providers` 字段，回显时丢失
   - 解决：commit `855f5c7` 补齐后端字段 + 前端回显逻辑

3. **多链余额数据格式不一致**
   - 原因：`mock_caw_client` 和 `real_caw_client` 返回的 treasury 结构不同，mock 返回单链，real 返回多链数组
   - 解决：统一 `TreasuryBalance` Pydantic 模型，新增 `chain` 和 `symbol` 字段，Dashboard 用 `reduce` 聚合多链显示
