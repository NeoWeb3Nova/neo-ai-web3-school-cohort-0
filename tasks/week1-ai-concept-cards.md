## 1. Large Language Model (LLM)

### 基本定义

LLM（大语言模型）通过 Transformer 架构，基于大规模语料进行预训练产生智能化。人工智能已发展多年，但真正被点燃的标志性事件是 ChatGPT 3.5 的发布。

### 核心机制

**(a) 向量空间**

在训练过程中，模型在空间里建立向量数据库，通过向量之间的空间关系来体现并产生智能。

**(b) 预测机制**

LLM 本质上是一种预测机制。当你输入一个词，它会预测下一个 token（词）。这种预测不是虚假的，而是含有真正的智能。

### 发展阶段与应用

| 阶段 | 特征 | 代表 |
|------|------|------|
| 早期阶段 | 以 Chat 为主，具有很强的聊天性质 | ChatGPT |
| 中期阶段 | 出现 Agent 和 Workflow 性质的应用 | 国内：豆包、Coze（扣子）；国外：N8N |
| 编程突破 | 大模型在编程领域特别强大，是最早的重大突破口 | AI Coding、Vibe Coding |
| 生产力阶段 | Agent（智能体）成为最热方向，给 LLM "插上手脚"之后能真正代替人干活 | 各类 Agent 框架 |

## 2. Prompt（提示词）

Prompt 是大模型的输入语料，也是用户与大模型沟通的桥梁。

### 演进阶段

**1. 早期阶段：重视 Prompt 工程化**

在模型发展初期，用户输入 Prompt 的好坏直接影响产出质量。掌握 Prompt Engineering 非常关键，需要用户有逻辑地编写，并遵循严格的格式化要求，才能让模型预测的 Token 更准确、更符合预期。

**2. 后期阶段：智能化与模糊化**

随着大语言模型的迭代：
- **输入门槛降低**：用户输入的信息可以比以前更模糊，不再需要把角色定义等细节说得面面俱到。
- **双向沟通**：如果模型不理解意图，它会通过反问与用户沟通，一步步引导用户输入提示词，从而得到最终信息。

## 3. Workflow（工作流）

Workflow 即"工作流"，通过逻辑链路和节点控制来接入大模型，并按照逻辑顺序来规范大模型的产出。

### 典型链路

1. **设定限制条件**：明确在什么情况下调用大模型。
2. **明确 Prompt**：规定大模型在特定条件下的提示词。
3. **格式化产出**：将大模型输出的信息进行格式化处理。
4. **数据抽取**：从格式化数据中提取关键信息，传递给下一个链路或节点使用。

通过这一系列逻辑控制，最终精准地实现用户需求。

## 4. Tool Use

当大模型发展到一定程度时，训练过程中赋予了它使用工具的能力。大模型在进行输入预测时，能够预测出它需要运行哪些工具，这种能力通常以"钩子"（Hooks）的形式出现。

### 给大模型"长出手脚"的手段

| 手段 | 说明 |
|------|------|
| MCP（Model Context Protocol） | 通用的命令行接口协议 |
| Skills | 将真实环境中的操作流程和步骤转化成 Skill 格式输送给大模型 |
| Bash / CLI | 通过命令行控制其他设备 |

### Skill vs MCP

- **Skill 与 MCP 是相辅相成的**，不能完全互相取代。
- **Skill 更像是一个 Workflow 或 SOP**：将真实环境中的操作流程和步骤转化成 Skill 格式输送给大模型，方便模型按照流程一步步执行。
- 虽然与之前的 Workflow 性质很像，但 Skill 是一种不断的演变，两者之间有很多相似点，也不完全一致。

## 5. AI Coding / Vibe Coding

**Vibe Coding** 概念最早由 Karpathy 提出。

### 为什么编程是大模型最强场景

1. 大模型在训练过程中包含了大量来自编程的语料。
2. 编程语言的逻辑性非常强。

在文本理解的基础上，大模型在代码层面上提供了非常强大的帮助，表现十分出色。

### 什么是 Vibe Coding

即使你不懂编程，也可以借助 AI，将你的需求以语音或文字的形式表达出来。大模型通过自身的理解来帮你编程实现，最终直接展现代码，或者通过代码的执行来帮你产生预期的结果。

## 6. Human in the Loop

因为大模型本质上是一种预测 Token 的机制，所以它的输出不一定完全正确，这时候就需要人类来把关。

### 把关的两个维度

**1. 内容准确性**

针对大模型产生的幻觉或不准确的答案进行校验。

**2. 安全防护（Guard）**

当大模型有了"手脚"、可以借助工具在本地电脑上运行时，就会涉及安全防护问题。

### 人类的核心作用：确权

- **确定风险可控**：在这种情况下，允许 AI 继续执行。
- **拒绝执行**：如果发现风险，直接拦截。

人类需要控制整体方向，而具体的执行步骤则交给 AI 来完成。

## 总结

| 概念 | 一句话理解 |
|------|-----------|
| LLM | 基于 Transformer + 大规模语料预训练的预测型智能系统 |
| Prompt | 用户与大模型沟通的输入桥梁，从"工程化"走向"智能化" |
| Workflow | 用逻辑链路和节点控制来规范大模型产出的工作流 |
| Tool Use | 大模型通过 Hooks 调用外部工具，"长出"手脚的能力 |
| AI Coding | 不懂编程也能借助 AI 生成可执行代码 |
| Human in the Loop | 人类在关键节点"确权"，把控方向与风险 |
