## 四、双赛道对比

### Cobo 赛道｜Agentic Economy × Cobo Agentic Wallet

**核心关注**：构建让 AI 参与经济活动的应用或基础设施。项目可以是一个能花钱、赚钱、交易的 Agent，也可以是让 Agent 经济运转的底层工具。

**5 个方向**：

| 方向 | 描述 | CAW 角色 |
|------|------|---------|
| 01 Agent-Native Payments | 让 Agent 成为互联网的一等支付公民：HTTP 402 自动完成支付，不依赖 API Key 和人类预注册 | 让 Agent 持有和支配资金 |
| 02 Trustless Agent Work Agreements | 基于 ERC-8183 实现 Agent 间去信任工作协议：发布→托管→交付→验收/驳回→付款 | Agent 资金账户，接收报酬、管理托管 |
| 03 Agent Resource Procurement | Agent 根据任务自主发现、比价、采购算力、数据、API、存储等资源 | 赋予 Agent 自主采购能力 |
| 04 Autonomous Trading | Agent 自主执行链上交易策略：套利、做市、组合管理、流动性挖矿 | 让 Agent 持有交易资金，授权边界内自主签署交易 |
| 05 A2A Economy | 多个 Agent 各自拥有钱包→Agent-to-Agent 经济体：互雇、拍卖、分账、管理 Treasury | 每个 Agent 独立 CAW 钱包，间直接支付、结算、分润 |

**赛道规则（硬性）**：
1. 项目必须围绕 Agent 与资金操作场景展开
2. Agent 的资金操作需通过 Cobo Agentic Wallet（CAW）完成
3. Agent 需具备**真实的资金执行能力**（支付、转账、结算、收益管理或资产调度），**非纯流程设计**
4. 项目需体现 CAW 在 Agent 钱包管理、权限控制、安全隔离或自主支付等方面的价值
5. 最终成果需为**可运行或可演示的产品原型（Demo）**，**不接受纯 PPT、概念设计或 Mockup**

**提交要求**：
- GitHub Repo
- README 与项目说明文档
- Demo 视频（建议 3–5 分钟）
- 项目演示链接（如有）
- 使用 CAW 的关键代码或配置说明
- 如涉及链上交互，建议提供：测试网地址、Transaction Hash、Agent Wallet 地址、流程截图或操作记录

**评审侧重点**：
1. **场景贴合度**：是否清楚体现 Agentic Commerce，AI 如何参与经济活动
2. **CAW 关键性**：CAW 是否是资金流程中的关键组件
3. **资金流程完整度**：Agent 从任务触发到资金操作完成的主要流程
4. **可演示性**：Demo 是否稳定展示核心流程
5. **风险边界说明**：权限、账户、链上交互或测试资产的基本边界和控制方式

### Z.AI 赛道｜Web3 × Long-Horizon Task

**核心关注**：GLM-5.1 的长程任务能力——让 Agent 不只是回答问题，而是能自主拆解复杂任务、制定多步骤计划、持续调用工具、迭代修复，完成从需求到交付的完整 Web3 工作流。

**3 个方向**：

| 方向 | 描述 |
|------|------|
| 01 Agentic Dev Tools for Web3 | 一句话交付完整 Web3 工程：需求分析、合约设计、代码编写、测试迭代、漏洞修复、部署准备与文档输出 |
| 02 AI-Powered 3D World Builder | 从空间规划、场景生成、资产配置到可部署包的完整生产链路（虚拟地产、NFT 展馆、游戏关卡、DAO 活动空间） |
| 03 AI × Creator Economy | 从创意输入、资料收集、创作、排版、发布到链上变现的完整内容生产链路 |

**赛道规则（硬性）**：
1. **推荐**使用 GLM-5.1 完成；Demo 中核心任务应由 GLM-5.1 驱动
2. 项目需围绕 Web3 × Long-Horizon Task，体现**长程、自主、持续执行**
3. Agent 需自主拆解复杂任务、制定多步骤计划、持续迭代/自我纠错
4. 完成有实质复杂度的 Web3 任务（开发、审计、空间设计、内容生产、链上发布、DAO 协作）
5. 最终成果需为**可运行或可演示的产品原型（Demo）**，**不接受纯 PPT、概念设计或 API Mock**

**提交要求**：
- GitHub Repo 与 README：说明项目目标、架构、运行方式、GLM-5.1 调用位置与关键流程
- 可运行 Demo 或可访问演示链接；建议提供 3–5 分钟 Demo 视频
- **长程任务运行记录**：展示 Agent 的任务拆解、工具调用、迭代/修复过程和最终交付成果
- Web3 相关证明：合约地址、测试网交易、NFT/链上资产、链上数据读写、钱包交互、协议集成记录或可部署 3D 场景包
- 边界说明：如涉及自动执行、权限、成本、外部工具、钱包、链上操作或自动发布，需说明安全边界、失败处理和人工介入条件

**评审侧重点**：
1. 赛道匹配度：Web3 × Long-Horizon Task（非普通问答/内容生成接口）
2. GLM-5.1 使用关键性：核心长程任务是否由 GLM-5.1 驱动
3. 任务复杂度与闭环程度：需求→计划→执行→验证→修复→交付完整链路
4. 长程稳定性：多步骤/多工具调用中保持目标一致
5. Web3 价值：解决 Web3 开发/3D设计/内容生产/链上资产/DAO协作的真实问题
6. 可演示性与可复现性
7. 安全、成本与权限边界

- 合约地址

- 测试网地址

- Agent Wallet 地址
