# Product

## Register

product

## Users

OPC operators and AI-agent owners who need to manage autonomous payment infrastructure. Primary users are technical founders and ops staff monitoring agent spending, issuing controlled payment cards, reviewing anomaly flags, and exporting audit trails. Context: high-stakes financial operations where a mistake means real money lost to a rogue agent or attack.

## Product Purpose

OPC Treasury is an Agent Finance OS that gives one-person companies programmable control over AI-agent spending. It issues scoped payment cards (CAW Pacts) with budget limits, vendor whitelists, time windows, and anomaly detection. The product prevents unauthorized spending before it happens, not after.

## Brand Personality

Precise. Guarded. Operational. The interface should feel like a mission-control dashboard — every number is trustworthy, every alert is actionable, nothing is decorative.

3-word personality: **technical, vigilant, sovereign**

## Anti-references

- SaaS cream/beige landing pages with big gradients and floating 3D illustrations
- Fintech apps that use navy-and-gold "trust" clichés
- Marketing copy like "supercharge", "unleash", "seamless"
- Decorative glassmorphism and blur-for-blur's-sake
- Dashboards that prioritize "beautiful" over "scannable at 2am"

## Design Principles

1. **Defense is the product** — The UI should make the policy engine's protection visible and comprehensible, not hidden behind abstractions.
2. **Show the gate, not just the garden** — Denied transactions, blocked attacks, and policy violations are first-class data, not footnotes.
3. **Agent autonomy with guardrails** — The interface must communicate both what an agent CAN do and what it CANNOT do, simultaneously.
4. **Operational density over marketing whitespace** — Every screen is a tool; information density is a feature when the user is monitoring multiple agents.
5. **Zero-trust visual language** — Status indicators, confirmations, and warnings must be unambiguous. Green means go, red means stop, orange means human attention required.

## Accessibility & Inclusion

- WCAG 2.1 AA minimum
- All status indicators use icon + color + text (never color alone)
- Reduced motion support for all animations
- High contrast for financial figures and threat indicators
