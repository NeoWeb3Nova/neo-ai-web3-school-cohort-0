## 第二幕：解决方案 — 虚拟员工卡 (0:33–1:33)

> **目标**: 1 分钟讲清楚"我们做了什么"和"为什么非 CAW 不可"

### 0:33 – 0:50 | 核心概念

| 项目 | 内容 |
|------|------|
| **画面** | 全屏展示 PactGuard 架构图（简洁动画版）：Neo (Owner) 在顶部 → CAW Service 在中间 → 三个 Agent (Content/Ad/Design) 在底部，每个 Agent 手里有一张发光的"卡" |
| **画面细节** | 卡片上飞出标签：月度预算 $200、单笔上限 $50、供应商白名单 [OpenAI, Midjourney]、冷却期 12h、时间窗口 00:00-23:59 |
| **旁白** | "PactGuard，基于 Cobo Agentic Wallet，为每个 AI Agent 发行一张虚拟员工卡。有预算、有上限、有白名单、有冷却期——Agent 能自主花钱，老板能睡安稳觉。" |
| **字幕** | 为每个 AI Agent 发行一张虚拟员工卡 |
| **BGM** | 电子乐渐入，节奏稳定，建立"技术感"和"安全感" |

### 0:50 – 1:10 | 协议层 — HTTP 402

| 项目 | 内容 |
|------|------|
| **画面** | 动画演示 x402 协议握手：① Agent 访问 OpenAI API → ② 服务器返回 HTTP 402 Payment Required → ③ Agent 查询 CAW Pact 预算 → ④ 在授权内签名 → ⑤ 带 Payment Proof 重试 → ⑥ 服务器验证 → ⑦ 返回结果 + Receipt。每个步骤用箭头连接，时序清晰 |
| **画面细节** | 在步骤 ③ 放大展示 Pact 策略检查（预算/白名单/时间窗口三个勾），步骤 ⑥ 放大展示 Facilitator 验证 |
| **旁白** | "我们在协议层集成了 HTTP 402——当 Agent 需要付费服务时，不再是人类预注册 API Key，而是 Agent 自己完成支付握手。预算够不够、供应商在不在白名单、时间窗口允不允许——CAW 在 MPC 签名前强制校验。" |
| **字幕** | HTTP 402 + CAW Pact = Agent 的一等支付能力 |
| **BGM** | 节拍加快，每个协议步骤对应一个鼓点 |

### 1:10 – 1:33 | 为什么非 CAW 不可

| 项目 | 内容 |
|------|------|
| **画面** | 左右分屏对比表，左侧"没有 CAW"（灰色、打叉），右侧"有 CAW"（亮色、打勾）：① 发预算：给私钥 vs 给 Pact API Key（泄露可吊销）；② 预算控制：客户端 if-else 可绕过 vs 服务端 MPC 强制校验；③ 白名单：Agent 被黑后乱转 vs CAW 只放行白名单地址；④ 审计日志：客户端可删除 vs 服务端 append-only |
| **画面细节** | 最后一条对比时，强调"去掉 CAW，整个系统崩溃"——左侧画面碎裂 |
| **旁白** | "没有 CAW，Agent 只能持有私钥自行支付，没有权限边界、没有审计、没有应急吊销。PactGuard 的每一道防线，都建立在 CAW 的服务端策略引擎上。去掉 CAW，系统直接崩溃。" |
| **字幕** | 去掉 CAW = 没有权限边界、没有审计、没有吊销 |
| **BGM** | 高潮前的蓄势，低音持续 |
| **转场** | 对比表收缩为 Dashboard 顶部的一个状态栏，镜头推进进入产品界面 |

> **0:03–0:33 痛点**
> "凌晨四点，Neo 又被他的 AI 员工吵醒了。给私钥？Agent 被攻击就是全盘归零。不给？业务直接停摆。月底对账，两百笔小微转账根本理不清。就像共享单车因为用户不是股东所以被破坏，AI Agent 因为没有财务边界所以随时可能失控。一人企业，需要一人企业的财务基础设施。"
>
> **0:33–1:33 方案**
> "PactGuard，基于 Cobo Agentic Wallet，为每个 AI Agent 发行一张虚拟员工卡。有预算、有上限、有白名单、有冷却期——Agent 能自主花钱，老板能睡安稳觉。我们在协议层集成了 HTTP 402——当 Agent 需要付费服务时，不再是人类预注册 API Key，而是 Agent 自己完成支付握手。预算够不够、供应商在不在白名单、时间窗口允不允许——CAW 在 MPC 签名前强制校验。没有 CAW，Agent 只能持有私钥自行支付，没有权限边界、没有审计、没有应急吊销。PactGuard 的每一道防线，都建立在 CAW 的服务端策略引擎上。"
>
> **1:33–2:33 正常流**
> "这是 PactGuard 的指挥中枢。三张活跃卡、275 美元已批准支出、1499 美元被拦截——异常不是隐藏数据，而是第一公民。在 Pact Manager 里，为每个 Agent 创建一张 Card Pact。审批一次，运行一月。Agent 每天自主运行。Content Agent 发现 OpenAI 余额不足，自动申请十美元。PactGuard 三秒完成三重检查——签名、上链、API 续费，一气呵成。老板还在睡觉，业务没断。"
