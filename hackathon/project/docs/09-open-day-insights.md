## 三、评审方式 — 启动会暴露的真实偏好

**1. 评委看什么：**

```
第一眼 → Demo (视频/演示)
第二眼 → 项目信息 (README / Report / 看板)
```

**2. 得分关键：**
- 能否在 **5 分钟内** 让评委印象非常深刻
- Demo 要 "做得漂亮、给人印象深刻"
- 若 Demo 出彩，即使其他文档简单，也有机会获奖

**3. 要求严格遵守：**
- 一个团队只能提交 **一个项目**
- 后台会审核，禁止"薵羊毛"式多提交/垄断

## 四、提交物详细要求

除了官网公开的 6 项必备提交物外，启动会主持人特别强调以下内容必须包含：

| # | 提交物 | 启动会原话 | 本项目对应行动 |
|---|---|---|---|
| 1 | Demo | "核心展示，尽量做得漂亮" | 视频要有 CLI 录屏 + 关键输出高亮 |

| 3 | 开发痕迹 | "已确定使用的 SDK/API/合约/测试网" | 在 README 专门一节列出已用的技术栈 |
| 4 | 任务看板 | "展示项目进度" | 在 GitHub Projects 或其他看板工具建一个 sprint 看板 |

## 八、给本项目的即时行动

1. **README 必须补充 4 点**：解决什么问题 / 目标用户 / 最小 MVP / 团队角色分工
2. **建一个任务看板** (GitHub Projects)＜评委会看
3. **Demo 视频不要超过 5 分钟**，要"印象深刻"——建议结构：开场 30s + 正常流 60s + 攻击演示 60s + 审计 30s
4. **专门列一节"已用技术栈"** — 列出 x402 / CAW / Base Sepolia / Python 等
5. **加入微信群**，获取助教支持，不要独自开发
6. **参加 Kamboo/Cobo Workshop** — 尽可能获取真实 SDK 和测试环境
7. **不要碰高风险边缘项目** — 这次是正规军，Polymarket 类项目会被扔
