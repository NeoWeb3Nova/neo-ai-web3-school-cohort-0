# 奖金池与评审解读

> 来源：启动会口述 + 官网规则
> 总金额：约 7000+ USDT（Cobo 3500 + Z.AI 3500，均含 API 补贴）

---

## 一、奖项设置

根据启动会：
- 奖项 "比较多"，希望更多团队能够获奖
- 具体名单 **下周公布**（未披露时以此为准）
- 除奖金外，优秀作品可能获得：
  - 项目方生态孵化机会
  - Boss 直聘对接

---

## 二、评审流程（启动会揭示的真实流程）

```
第一步：看 Demo（最重要）
  能否在 5 分钟内让人印象深刻？
    ↓ Yes
第二步：看 README / Report / 看板
  解决什么问题？目标用户？最小 MVP？团队角色？
    ↓ Yes
第三步：综合评分
  + 赛道匹配度
  + CAW/GLM-5.1 关键性
  + 可演示性
  + 创新性
```

---

## 三、得分策略

### 高权重（必须拿满）

| 维度 | 权重 | 如何拿满分 | PactGuard 对应 |
|---|---|---|---|
| 场景贴合度 | 高 | 清楚体现 Agentic Commerce | "让 Agent 成为一等支付公民" |
| CAW 关键性 | 高 | CAW 是资金流程核心组件 | Pact 校验是支付前必须步骤 |
| 资金流程完整度 | 高 | 从任务触发到链上结算完整 | 请求→评估→签名→结算→审计 |
| 可演示性 | 高 | 5 分钟内稳定展示核心流程 | Demo 视频严格控制 5 分钟 |

### 中权重（争取加分）

| 维度 | 权重 | 加分项 | PactGuard 对应 |
|---|---|---|---|
| 风险边界说明 | 中 | 权限、账户、测试资产边界 | 专门 "Risks & Boundaries" 章节 |
| 创新性 | 中 | 8 种攻击场景全覆盖 | 威胁模拟器 |
| 执行力 | 中 | 从 idea 到 Demo 的完成度 | 两周完成架构+代码+文档+攻击模型 |

---

## 四、制胜策略

1. **Demo 第一**：视频是评委最先看的东西，必须"漂亮、给人印象深刻"
2. **不要堆技术**：用故事线而不是技术文档打动评委
3. **展示壁垒**：为什么是我们做，而不是别人？（行业理解 + 安全深度）
4. **时机叙事**：x402 刚发布、Agent 经济趋势刚起——timing 就是壁垒

---

## 五、注意事项

- 一个团队 **只能提交一个项目**（后台审核，禁止薵羊毛）
- 不要暗向同一赛道，获奖概率会降低
- 两周都要认真开发：第一周分享多、有助教；第二周专注 Demo
