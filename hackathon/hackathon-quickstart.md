## 二、赛道选择决策树

### 赛道对比

| 维度 | ZI 赛道 | Kamboo (Cobo) 赛道 |
|------|-----------|---------------------|
| 核心方向 | AI 模型应用 / 开发者工具 | 钱包 / 支付 / 资产管理 |
| 关键能力 | 长程任务执行 (GLM-5.1) | Agent 资金操作 (CAW) |
| 技术栈 | GLM-5.1, DevPack, API | Cobo Agentic Wallet SDK |
| 奖金池 | 3,500 USDT + API 补贴 | 3,500 USDT |
| 适合人群 | 熟悉 LLM 调用、工程化 | 熟悉 Web3 底层、安全模型 |

### 选择建议

1. **不要盲目扎堆** — 主办方下周会公布各赛道报名数量，人多的赛道获奖概率降低
2. **一个团队只能提交一个项目** — 后台会审核，禁止多作品蒕羊毛
3. **根据自己能力圈选择** — 会 LLM 调用选 ZI，会钱包/合约安全选 Kamboo
4. **不强制使用赞助商 SDK** — 但需满足赛道方向要求，具体偏好建议在各赛道 Workshop 上直接问寻道方

## 三、提交物清单（必须项）

### 核心交付物

```
├── Demo / 演示视频（建议 3–5 分钟，突出核心流程）
├── GitHub Repo
│   ├── README：项目目标、安装/运行方式、技术架构、API/SDK 使用说明
│   ├── 核心代码（尤其是使用赞助商 SDK 的关键代码段）
│   └── 可运行原型 / 演示链接
├── 项目说明文档
│   ├── 解决什么问题？
│   ├── 目标用户是谁？
│   ├── 最小 MVP 是什么？
│   ├── 团队角色分工
│   └── 当前完成度与后续计划
└── 任务看板（展示开发进度）
```

### 评审看什么

| 优先级 | 内容 | 备注 |
|--------|--------|--------|
| P0 | **Demo** | 首先看 Demo，可运行原型是硬性要求，不接受纯 PPT/Mockup |
| P1 | 项目信息 | README、Report、看板——看团队是否清晰思考 |
| P2 | 印象分 | 5 分钟内让人印象深刻 → 得分更高 |

## 四、Demo 打磨要点

### 时长控制

- **目标**: 尽量在 **5 分钟内** 完成核心演示
- **原则**: 突出一个最亮的点，而非面面俰到

### 结构建议

1. **开场 30 秒** — 一句话说清楚解决什么问题
2. **问题展示 1 分钟** — 让观众感受痛点
3. **解决方案 2 分钟** — 核心交互流程
4. **技术亮点 1 分钟** — 展示 AI/CAW 的关键作用
5. **收尾 30 秒** — 团队介绍 + 感谢

### UI 建议

- 界面尽量做得漂亮、给人印象深刻
- 演示流畅，提前反复练习
