> 精炼版，面向评委与队友的快速阅读。
> 完整版见 `experiments/x402-caw-agent-payment-loop/risks.md`

## 项目概述

PactGuard 为 AI Agent 自主支付建立程序化约束层。赛前 (6/12 截止) 的核心目标是展示"在 Pact 保护下的端到端支付 Demo"。本 Memo 记录可能影响 Demo 完成度的风险及应对策略。

---

## 技术依赖风险 (P0 — 可能阻断 Demo)

### R1: x402 Python SDK 未发布

| 项 | 内容 |
|---|---|
| 可能性 | **高** |
| 影响 | 无法发起真实 402 请求，服务端不可用 |
| Fallback | 内置 `MockServer` 类，模拟 x402 402 响应、Payment-Receipt、idempotencyKey 去重 |
| 当前状态 | ✅ MockServer 已完成，自包含运行通过 |
| 评委说明 | 代码结构已预留 `x402.Client` 接入点，SDK 发布后替换一行即可 |

### R2: CAW SDK 未发布 / 无测试环境

| 项 | 内容 |
|---|---|
| 可能性 | **高** |
| 影响 | 无法调用真实 Pact 检查 API |
| Fallback | 模拟 `CoboCAWWallet` 类，接口与预期 SDK 一致 (`check_pact`, `sign_payment`, `log_action`) |
| 当前状态 | ✅ v1/v2 双版本 `check_pact()` 已完成，含 13 项检查 |
| 评委说明 | Pact 逻辑已完整实现，SDK 发布后替换为真实调用即可 |

### R3: Base Sepolia USDC 水龙头限额

| 项 | 内容 |
|---|---|
| 可能性 | **中** |
| 影响 | 无法获取测试网资金，真实链上交易不可行 |
| Fallback | Anvil 本地链，或完全不上链（模拟合约交互 + 模拟 txHash） |
| 当前状态 | 🔄 待申请 |
| 评委说明 | Demo 展示的是"签名前拦截"逻辑，链上结算为次要验证层 |

## 安全风险 (P1 — Demo 展示核心)

### R4: 8 种攻击场景拦截验证

| 攻击 | v1 拦截 | v2 拦截 | 风险 |
|---|---|---|---|
| A1 Prompt Injection → 越权转账 | ✅ | ✅+warn | 技术层面已覆盖 |
| A2 伪造返回 → 超额定价 | ✅ | ✅ | 技术层面已覆盖 |
| A3 范围绕过 → 未知合约 | ✅ | ✅+human_review | 技术层面已覆盖 |
| A4 黑名单函数 → self_destruct | ✅ | ✅+blocked | 技术层面已覆盖 |
| A5 预算耗尽 → 高频小额 | ❌ | ✅ (累计+频率) | **关键差异** |
| A6 时间窗口绕过 | ✅ | ✅ (双重检查) | 技术层面已覆盖 |
| A7 重放攻击 | ✅ | ✅ | 技术层面已覆盖 |
| A8 社会工程 | ❌ | ❌ | **技术盲点，需治理层** |

**关键结论**：v2 相对 v1 的核心提升是**状态化防御**（累计预算、小时频率、连续失败暂停）。A8 社会工程是所有纯技术方案的盲点，需在 README 中明确标注为 "Governance Layer / UI 层防护"。

## 商业风险 (P2 — 不影响 Demo，影响长期)

| 风险 | 影响 | 缓解 |
|---|---|---|
| 交付物质量差 | Agent 付款后内容不可用 | Escrow 模式 (ERC-8183) + 小额试点 |
| 服务方跑路 | 收款不交付 | 声誉系统 (ERC-8004) + Escrow 超时退款 |
| 争议无法仲裁 | 质量主观分歧 | 量化验收标准 + 第三方 Evaluator |

> 当前 Demo 范围不包含 Escrow 完整实现，但架构文档已预留接口。

## 执行风险 (P1 — 单人参赛)

| 风险 | 影响 | 缓解 |
|---|---|---|
| 工程量过大 | 10 天内无法完成全部愿景 | 范围锁定"可演示的最小闭环"，放弃 UI、多链、真实后端 |
| 队友未找到 | 全栈一人承担 | 优先保证 CLI Demo + 文档，UI 为 Nice-to-have |
| 健康/时间突发 | 最后几天无法工作 | 6/10 前完成核心，预留 2 天缓冲 |
